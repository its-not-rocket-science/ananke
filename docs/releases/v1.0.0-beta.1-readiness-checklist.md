# Ananke v1.0.0-beta.1 — Beta Launch Readiness Checklist

> Gate owner: Release Manager / QA Lead  
> Status source of truth: `.github/beta-launch-readiness.json`
> Hard release gate: `.github/workflows/beta-launch-readiness.yml` (`release-gate` must be green)

## Review protocol (required before checking any box)

1. Run/inspect the matching CI job.
2. Attach concrete evidence (run URL, report, screenshot, or document link) in `.github/beta-launch-readiness.json`.
3. Add reviewer initials + date in the evidence artifact.
4. Only then mark checklist item as complete.

---

## 1) Technical readiness (must all pass)

- [ ] **All tests pass on Node 18/20/22, Windows/macOS/Linux**
  - Validation method: GitHub Action `beta-launch-readiness.yml` → `technical-matrix`.
  - Required evidence: workflow run URL + matrix screenshot.
- [ ] **Determinism fuzzer: 10,000 / 10,000 seeds pass**
  - Validation method: `npm run test:determinism:nightly` in `determinism-fuzzer` job.
  - Required evidence: run URL + seed count output.
- [ ] **Performance benchmarks meet budget** (`.github/perf-budget.json`)
  - Validation method: `npm run benchmark:run && npm run benchmark:budget` in `performance-budget` job.
  - Required evidence: generated benchmark report + budget check output.
- [ ] **No memory leaks (48-hour soak test)**
  - Validation method: soak profile over 48h using production-like workload.
  - Required evidence: memory trend chart + analysis write-up linked in `.github/beta-launch-readiness.json`.
- [ ] **WASM backend loads and passes determinism suite**
  - Validation method: `npm run test:wasm` and wasm determinism suite in `wasm-determinism` job.
  - Required evidence: wasm test logs + determinism diff output.

## 2) API stability (must all pass)

- [ ] **Tier 1 exports frozen (API diff since v0.1.69 = empty)**
  - Validation method: `api-stability` job (Tier 1 surface check + diff report).
  - Required evidence: attached API diff artifact showing zero changes.
- [ ] **All Tier 1 functions have JSDoc examples**
  - Validation method: API docs generation + doc QA review.
  - Required evidence: review checklist and spot-check links.
- [ ] **Deprecation policy documented**
  - Validation method: doc review of `docs/api-lifecycle.md`.
  - Required evidence: approval note from maintainers and SDK owners.

## 3) Documentation (must all pass)

- [ ] **4 learning paths complete and tested by fresh user**
  - Validation method: guided run-throughs of each path by first-time user.
  - Required evidence: 4 completion logs and feedback notes.
- [ ] **API reference generated and searchable**
  - Validation method: `npm run docs:api` + docs search smoke test.
  - Required evidence: generated docs commit + search query screenshots.
- [ ] **Video tutorial published (YouTube)**
  - Validation method: public YouTube upload with final title/description.
  - Required evidence: video URL + publish timestamp.

## 4) Adoption readiness (must all pass)

- [ ] **3 external users have built prototypes (signed NDA or public)**
  - Validation method: external prototype verification (public repos or NDA records).
  - Required evidence: links/IDs + one-line summary per prototype.
- [ ] **0 P1 bugs reported in last 14 days**
  - Validation method: issue tracker query filtered to P1 + last 14 days.
  - Required evidence: query URL/screenshot with timestamp.
- [ ] **Support channel (Discord) has 10+ active members**
  - Validation method: Discord analytics export.
  - Required evidence: member activity screenshot/report link.

## 5) Legal & governance (must all pass)

- [ ] **LICENSE file correct (MIT/Apache 2.0)**
  - Validation method: legal review and repository file check.
  - Required evidence: legal signoff entry.
- [ ] **CLA signed by all contributors (if required)**
  - Validation method: CLA system export against current contributor list.
  - Required evidence: signed contributor report link.
- [ ] **Security policy published**
  - Validation method: `SECURITY.md` presence and policy review.
  - Required evidence: security lead approval note.

## Final release decision

**Do not announce `v1.0.0-beta.1` unless all boxes are checked, `.github/beta-launch-readiness.json` fields are satisfied, and `Beta Launch Readiness / release-gate` is green.**
