# Security Policy

## Supported Versions

Only the latest **major** release line is supported for security fixes.

| Version | Supported |
| ------- | --------- |
| Latest major (e.g. `v3.x`) | ✅ |
| Older majors | ❌ |

## Reporting a Vulnerability

Please report vulnerabilities privately.

### Preferred disclosure path

1. Open a private GitHub Security Advisory draft:
   - Repository → **Security** → **Advisories** → **Report a vulnerability**.
2. Include clear reproduction steps, severity estimate, and impacted versions.

### Backup disclosure path

- Email: **security@its-not-rocket-science.dev**

If you email us, please also include your GitHub username so we can coordinate via Security Advisories.

## Response SLA

- **P1 (critical/high):** initial triage and response within **7 calendar days**.
- **P2 (medium):** initial triage and response within **30 calendar days**.

We will keep reporters informed throughout triage, remediation, and coordinated disclosure.

## Disclosure and credit

- We prefer coordinated disclosure after a patch is available.
- We credit reporters in advisories/release notes unless anonymity is requested.
